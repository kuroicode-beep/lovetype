# LoveType 기능 리스트

> 버전: 1.2.1 | 최종 점검: 2026-03-13

---

## 1. 화면 구조

| 화면       | ID                  | 진입 조건                              |
|------------|---------------------|----------------------------------------|
| 랜딩 화면  | `#landing-screen`   | 초기 진입, 오늘 미완료 상태            |
| 문항 화면  | `#question-screen`  | 테스트 시작 버튼 클릭                  |
| 결과 화면  | `#result-screen`    | 20문항 응답 완료 또는 오늘 완료 이력 있을 때 |

화면 간 전환은 `hidden` 속성 토글 방식으로 처리하며, URL은 변경되지 않는 원페이지 구조입니다.

---

## 2. 랜딩 화면 기능

### 2-1. 로딩 및 초기화
- 앱 진입 시 4개 JSON 데이터 파일(`lovetype_questions.json`, `base_result.json`, `axis_modifier.json`, `share_copy.json`)을 자동으로 로드
- 로드 중에는 시작 버튼을 비활성화 상태(`aria-disabled`, `aria-busy`)로 유지
- 로드 완료 시 시작 버튼 활성화
- 로드 실패 시 실패 파일 이름과 원인을 안내 문구로 표시
- `file://` 프로토콜로 실행할 경우 별도 안내 문구 표시
- 문항 수(`totalQuestions`)를 데이터 파일 기반으로 화면에 반영

### 2-2. 히어로 영역
- 서비스 캐치프레이즈, 부제 카피 표시
- 테스트 정보 메타 배지(문항 수, 4지선다, 축 분산 셔플, 수평 꺾은선 결과) 표시
- 테스트 시작 CTA 버튼

### 2-3. 비주얼
- CSS 기반 심볼 애니메이션 (orbit-a, orbit-b, 하트 아이콘, 스파크 포인트)
- 별도 이미지 에셋 없이 순수 CSS로 구성

---

## 3. 테스트(문항) 화면 기능

### 3-1. 문항 셔플 및 축 분산
- 4축(EI, SN, TF, JP) 각 5문항을 축 간 연속 출력이 최소화되도록 분산 배열
- 출력 전 축별 문항 순서도 랜덤 셔플
- 각 문항의 선택지(4지선다)도 개별 랜덤 셔플

### 3-2. 문항 렌더링
- 현재 문항 번호 및 전체 문항 수 표시 (`1 / 20` 형식)
- 진행률 바(progress bar) 실시간 업데이트
- 축 코드 및 축 이름 표시 (예: `EI · 관계 에너지`)
- 문항 테마 표시 (예: `첫 흐름`)
- 문항 본문 텍스트 표시
- 선택지 4개를 카드 버튼으로 렌더링
- 화면 전환 시 첫 번째 선택지 버튼에 자동 포커스

### 3-3. 답변 처리
- 선택지 클릭 즉시 응답 저장 후 다음 문항으로 이동
- 마지막 문항 응답 시 결과 계산 단계로 전환

---

## 4. 결과 계산 기능

### 4-1. 축 점수 집계
- 4개 축(EI, SN, TF, JP) 각각 응답 값(E/I, S/N, T/F, J/P)을 카운트
- 각 축의 우세 방향 결정

### 4-2. 강도 판정
| 점수 분포 | 강도       |
|-----------|------------|
| 5:0       | strong     |
| 4:1       | strong     |
| 3:2       | weak       |

### 4-3. 그래프 값 산출
| 강도   | 절댓값 |
|--------|--------|
| strong_5_0 | 100 |
| strong_4_1 | 72  |
| weak       | 38  |

우세 방향이 각 축의 양수 기준 글자(E, S, T, J)면 양수, 반대면 음수로 처리

### 4-4. MBTI 코드 산출
- 4축 우세 방향 조합으로 4글자 코드 결정 (예: `INFP`)

### 4-5. 캐시 키 생성
- `{MBTI}__{EI 방향_강도}__{SN 방향_강도}__{TF 방향_강도}__{JP 방향_강도}__v1` 형식

### 4-6. 결과 텍스트 조합
- `base_result.json`에서 MBTI 유형별 기본 텍스트(`summary`, `base_description`, `relationship_example`, `closing_guidance`) 로드
- `axis_modifier.json`에서 4축 × 강도 조합 수식어 텍스트 로드
- `share_copy.json`에서 공유 템플릿 로드, `{mbti}` 및 `{summary}` 플레이스홀더 치환
- 데이터 파일 로드 실패 시 자체 fallback 텍스트 사용
- 결과 필드에 `?` 포함 시 fallback 텍스트로 대체 처리

### 4-7. API 연동 및 로컬 폴백
- `/api/mbti-test/result` 엔드포인트로 POST 요청
- API 실패 시 로컬 계산 결과로 자동 대체 (`local-fallback` 모드)

## 5. 결과 화면 기능

### 5-1. 결과 카드 (capture-card)
- MBTI 4글자 코드 표시
- 4축 강도 요약 텍스트 표시 (예: `관계 에너지 I weak (내향) / ...`)
- 결과 요약 문구 표시
- 캐시 키 표시

### 5-2. LoveType 흐름 그래프
- 4축(EI → SN → TF → JP)을 x축으로, 그래프 값(-100 ~ +100)을 y축으로 한 수평 꺾은선 그래프
- 양수는 `그래프 긍정색(pink)`, 음수는 `그래프 부정색(mint)`으로 점 구분
- 가이드라인(Strong +, Weak +, 균형, Weak -, Strong -)과 중앙선 표시
- 각 점에 `방향 · 축 이름 · 강도` 텍스트 레이블 표시
- 하단에 4축별 상태 카드(강도, 상태 라벨) 표시
- 상태 라벨: 선명 / 기울어짐 / 잔잔 / 깊음

### 5-3. LoveType 해석
- 기본 설명 단락 (`base_description` + EI 수식어 + SN 수식어)
- 연애 예시 단락 (`relationship_example` + TF 수식어 + JP 수식어)
- 마무리 단락 (`closing_guidance` + 강도 조합 키 텍스트)

### 5-4. 공유 문구
- 유형별 공유 템플릿 문구 표시

### 5-5. 결과 액션 버튼
| 버튼              | 동작                                 |
|-------------------|--------------------------------------|
| 더 많은 연애 이야기 보기 | `#stories` 앵커 이동 (미구현 섹션)   |
| 공유하기          | 공유 기능 실행                       |
| 결과 다시 보기    | `#result-screen` 앵커 이동           |

## 6. 공유 기능

- `navigator.share` 지원 환경: 네이티브 공유 시트 호출
- 미지원 환경: 클립보드에 공유 문구 + 캐시 키 복사 후 알림
- 클립보드도 불가한 경우: `alert`으로 공유 텍스트 표시

## 7. 상태 저장 및 재방문 처리

### 7-1. localStorage 저장 항목

| 키                          | 내용                         |
|-----------------------------|------------------------------|
| `lovetype_result_payload`   | 전체 결과 JSON               |
| `completed_date`            | 완료 날짜 (KST 기준)         |
| `mbti_result`               | MBTI 코드                    |
| `axis_scores`               | 4축 점수                     |
| `axis_strength`             | 4축 강도 (flat 형식)         |
| `graph_values`              | 4축 그래프 값                |
| `cache_key`                 | 캐시 키                      |
| `result_version`            | 앱 버전                      |
| `result_summary`            | 요약 문구                    |
| `result_detail`             | 전체 결과 텍스트 JSON        |
| `next_available_at`         | 다음 가능 일시 (KST 자정 기준 UTC) |

### 7-2. 재방문 처리
- 저장된 결과의 `result_version`이 현재 앱 버전과 다르면 무효 처리
- 오늘 이미 완료한 경우 랜딩 화면 진입 시 결과 화면으로 바로 이동
- 재방문 완료 안내 문구 표시

## 8. 테마 기능

### 8-1. 기본 테마 / 고대비(HC) 테마
- `data-theme=""` (기본) / `data-theme="hc"` (고대비) 전환
- CSS 변수 오버라이드로 색상 전체 교체
- 기본 테마: 핑크·코랄·피치 계열
- 고대비 테마: 민트·옐로우·다크 계열

### 8-2. 테마 저장 및 복원
- `localStorage`의 `theme_mode` 키로 저장
- URL 파라미터 `?mode=hc`로 강제 고대비 모드 진입 가능
- 페이지 로드 시 저장값 또는 URL 파라미터 기준으로 테마 자동 복원

### 8-3. 토글 UI
- 헤더 우측에 토글 스위치 배치
- `role="switch"` + `aria-checked` + `aria-labelledby` 접근성 적용

---

## 9. 접근성

- 페이지 최상단 Skip to content 링크
- 모든 인터랙티브 요소에 `aria-label` 또는 `aria-labelledby` 적용
- 스크린 리더용 `role="status"` + `aria-live="polite"` 로드 상태 알림
- `role="group"` + `aria-labelledby`로 선택지 그룹 마크업
- `:focus-visible` 포커스 스타일
- SVG 그래프에 `aria-hidden="true"` 처리 (텍스트 대체 설명으로 보완)
- `[hidden]` 속성을 `display: none !important`와 함께 사용

| 브레이크포인트 | 주요 변화                              |
|----------------|----------------------------------------|

## 11. 버전 관리 및 캐시 제어

- CSS, JS 파일에 `?v=1.2.1` 쿼리스트링으로 캐시 버스팅
- `APP_VERSION = "1.2.1"` 상수로 버전 일관성 유지
- `localStorage` 저장 결과에 `result_version` 포함하여 버전 불일치 시 자동 무효화
